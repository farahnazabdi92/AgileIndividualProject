AI Reflection Log

This page documents AI-assisted work, evaluates reliability, and records corrections.

Purpose and Context

This log documents the real AI-assisted workflow used in the SkillTrack Learning Portfolio. It distinguishes how ChatGPT and Codex were used, explains the phase-driven approach, and records the review and acceptance checks applied before moving forward. The goal is to show responsible use, not delegation of understanding.

All AI outputs were reviewed against the assignment brief and acceptance criteria before being accepted.

AI Use Case 1

Use Case: Understanding Context and Requirements with ChatGPT

Prompt focus: Understand the brief and rubric, extract explicit requirements, and build a checklist to avoid missing marks.

What the AI produced

  • A structured list of requirements for each page and feature.
  • Rubric expectations grouped into basic vs excellent bands.
  • Early risk flags on high-impact items like the quiz and demo explainability.

What was changed or refined

  • Removed inferred items that were not explicitly in the brief.
  • Reordered requirements to match the intended user journey.
  • Condensed overlap so the checklist stayed practical and usable.

Why it helped: It accelerated the initial analysis, but human review was needed to keep the scope aligned to the brief.

Limitations noted: Early drafts were too broad and needed pruning to avoid scope creep.

AI Use Case 2

Use Case: Phase Planning and Prompt Design with ChatGPT

Prompt focus: Break the project into sequential phases and draft phase-specific prompts for Codex.

What the AI produced

  • A 12-phase development roadmap aligned to the assignment.
  • Phase prompts that specified scope and non-negotiable requirements.
  • Checklists and acceptance criteria for each phase.

What was changed or refined

  • Rewrote prompts to remove ambiguity and align with the rubric.
  • Added explicit "do not implement yet" constraints per phase.
  • Adjusted the phase order to keep dependencies under control.

Why it helped: Phase planning reduced complexity and made each step verifiable before moving forward.

Limitations noted: Some prompts were too generic and required manual tightening to match the assignment scope.

AI Use Case 3

Use Case: Phase-by-Phase Implementation with Codex

Prompt focus: Implement each phase in the repository, then review and correct the output before moving on.

What the AI produced

  • Shared UI shell, page scaffolds, and consistent navigation.
  • Tutorial content, quiz rendering, and storage logic.
  • CSS theming updates and responsive layout refinements.

What was changed or refined

  • Adjusted content depth to match rubric expectations.
  • Refined validation behaviour and reward handling.
  • Standardized wording and cross-page consistency.

Why it helped: Codex accelerated implementation, but every phase required manual verification to ensure the work was explainable and compliant.

Limitations noted: Some implementations looked correct at first glance but still needed manual review against acceptance criteria.

12-Phase AI-Assisted Development Workflow

The project was intentionally split into 12 phases so each step could be planned, implemented, and reviewed before moving on. ChatGPT helped structure the phases and refine prompts, while Codex implemented each phase in the repository. This reduced complexity and kept every change aligned with the assignment requirements.
