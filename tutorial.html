Tutorial

This learning module will guide users through HTML, CSS, and JavaScript fundamentals in a structured, supportive way.

Introduction and Context

This tutorial is the learning core of SkillTrack. It introduces the essential building blocks of the web so you can recognise how structure, presentation, and behaviour work together in a single product. The goal is not just to define terms but to show how they are applied in clear, readable code you can explain.

You will move from structure (HTML) to styling (CSS) and finally to interaction (JavaScript), then apply those ideas in a short interactive demo.

Learning Objectives

By the end of this page you should be able to:

  • Explain the distinct roles of HTML, CSS, and JavaScript.
  • Identify semantic HTML elements and describe why they matter.
  • Use basic CSS selectors and properties to control appearance.
  • Describe how JavaScript responds to events to change a page.
  • Connect these ideas when reviewing a small interactive example.

HTML Fundamentals

HTML provides the structure of a page. It describes what the content is, not how it should look. Semantic elements such as <header>, <main>, and <footer> make that structure clear for both humans and tools such as screen readers.

Example: A Simple Semantic Structure

<header>
  <h1>SkillTrack Learning Portfolio</h1>
  <nav>
    <a href="tutorial.html">Tutorial</a>
    <a href="quiz.html">Quiz</a>
  </nav>
</header>

<main>
  <section>
    <h2>HTML Fundamentals</h2>
    <p>Structure comes first.</p>
  </section>
</main>

<footer>
  <p>References listed on the CV page.</p>
</footer>

This structure tells the browser and the reader what each region means, which improves clarity and accessibility.

Key Ideas

  • Use heading levels in order to show a clear content hierarchy.
  • Use attributes like href and alt to add essential meaning.
  • Prefer semantic tags over generic containers when you know the purpose.

CSS Fundamentals

CSS controls presentation. It selects HTML elements and applies rules that change layout, color, spacing, and typography. Keeping CSS separate from HTML makes it easier to maintain and reuse styles across a site.

Example: Class Selectors and Spacing

/* A class selector targets any element with class="card" */
.card {
  border: 1px solid #d7dde4;
  padding: 1rem;
  border-radius: 12px;
}

/* Utility classes create consistent spacing */
.mt-2 {
  margin-top: 0.5rem;
}

Class selectors help you reuse styles without repeating inline formatting on every element.

Key Ideas

  • Element selectors apply to every matching tag; class selectors are more precise.
  • Spacing uses margin (outside) and padding (inside) to control layout.
  • Consistent naming makes large stylesheets easier to read.

JavaScript Fundamentals

JavaScript adds behaviour. It lets the page respond to user actions such as clicks or form input. A common pattern is to listen for an event and then update the page dynamically.

Example: Responding to a Click

// Find the button and the message
const button = document.querySelector(".demo-btn");
const message = document.querySelector(".demo-status");

// Update the message when the button is clicked
button.addEventListener("click", () => {
  message.textContent = "Style updated!";
});

Events connect user actions to page changes, which is the foundation of interactivity.

Key Ideas

  • Variables store values you want to reuse later.
  • Functions group actions into reusable steps.
  • Event listeners run code in response to user behaviour.

Best-Practice Rationale

            

                Good web development is about clarity and maintainability. The choices below make your work easier
                to understand, test, and extend.
            

            
    
                
  • Use semantic HTML so the structure communicates meaning.
    • 
                
  • Keep CSS separate from HTML to avoid repeating styles everywhere.
    • 
                
  • Name classes consistently so styles are easy to reuse.
    • 
                
  • Keep JavaScript focused on behaviour and keep functions readable.
